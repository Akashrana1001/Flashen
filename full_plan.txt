    FLASHEN FULL PLAN
    Date: 2026-04-18
    Project: Flashen - AI-Assisted Flashcard Engine

    ==================================================
    1) PROJECT GOAL
    ==================================================
    Build and scale a high-retention learning system that helps students convert study material into spaced-repetition flashcards, practice efficiently, and improve long-term memory with data-driven feedback.

    Primary objective:
    - Improve retention, consistency, and exam readiness through AI ingestion + SM-2 scheduling + analytics.

    Secondary objective:
    - Provide smooth, reliable, secure, and responsive experience across devices.

    ==================================================
    4) MAJOR PROBLEMS, SOLUTIONS, AND EXECUTION PLAN
    ==================================================

    Problem 1: Learners forget material quickly after one-time study.
    Solution:
    - Use SM-2 spaced repetition scheduling for scientifically timed reviews.
    Execution:
    - Keep grade-based update logic robust and monitored.
    - Surface due cards clearly in dashboard/practice pages.
    Expected impact:
    - Strong reduction in forgetting curve loss across 2-6 weeks.

    Problem 2: Manual flashcard creation is slow and causes friction.
    Solution:
    - AI-powered PDF ingestion creates flashcards automatically.
    Execution:
    - Improve prompt structure and output validation.
    - Add post-generation refinement controls.
    Expected impact:
    - Faster time-to-first-practice and higher onboarding completion.

    Problem 3: Students cannot see what to improve next.
    Solution:
    - Provide mastery analytics, retention trends, and due forecasts.
    Execution:
    - Enhance chart clarity and insight wording.
    - Add weak-topic recommendations.
    Expected impact:
    - Better study decisions and higher review consistency.

    Problem 4: High input frequency can break session stability in practice mode.
    Solution:
    - Harden state transitions and mutation handling against rapid keyboard events.
    Execution:
    - Add input throttling/debouncing safeguards where needed.
    - Keep E2E stress scenario for rapid key actions.
    Expected impact:
    - Fewer session glitches and improved learner trust.

    Problem 5: Large files or unreliable AI responses can fail ingestion.
    Solution:
    - Graceful error states, progress feedback, retry paths.
    Execution:
    - Add robust failure messaging and optional partial-result fallback.
    - Validate max file limits and parser boundaries.
    Expected impact:
    - Lower abandonment during upload/ingestion.

    Problem 6: Session expiration can cause confusing redirects.
    Solution:
    - Smooth auth guard behavior and clear session-expired messaging.
    Execution:
    - Standardize token checks across protected routes.
    - Show explicit, user-friendly re-login prompt.
    Expected impact:
    - Reduced confusion and support burden.

    Problem 7: Traffic spikes can degrade API reliability.
    Solution:
    - Load-tested backend and tuned rate limits.
    Execution:
    - Repeat K6 performance runs by release.
    - Monitor p95 latency and error-rate thresholds.
    Expected impact:
    - Stable response times and fewer downtime incidents.

    Problem 8: Mobile screens can break dense learning layouts.
    Solution:
    - Responsive patterns for dashboard/library and horizontal scroll fallbacks.
    Execution:
    - Run recurring mobile manual QA checks.
    - Prioritize accessibility and tap-target quality.
    Expected impact:
    - Better mobile adoption and session duration.
